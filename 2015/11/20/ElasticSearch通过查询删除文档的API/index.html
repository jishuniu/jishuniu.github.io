请求主体

通过查询的删除可以在它的主体内使用Query DSL以表示这个查询应该被执行并删除所有的文档。主体内容也可以是命名为source的REST参数。

分发

通过查询的删除接口从所有的主分片散发出去，并从那里复制所有分片副本。

更新路由

路由值（这些值为用逗号隔开的列表）可以被指定以控制该分片由查询请求来执行删除。

限制

通过查询的删除不支持以下查询和过滤器： has_child，has_parent 和 top_children。
